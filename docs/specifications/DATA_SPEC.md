# Data Specification

## Dataset

| Field | Value |
|-------|-------|
| Name | Bone Fracture X-Ray Images (expanded dataset) |
| Source | `data/archive/Bone_Fracture_Dataset` (organized into train/test splits) |
| Local path | `data/train/` and `data/test/` |
| Total images | 9,232 (8,726 train + 506 test) |
| Format | JPG, JPEG, PNG |
| Image dimensions | 1560 × 1170 pixels (loaded at native size) |

## Description

A binary classification dataset of X-ray images organized into train and test splits, each with two class folders. The original 90-image Colab dataset is archived under `data/archive/`. Results should be interpreted as educational, not clinically validated.

## Class Labels

| Label | Folder | Index (alphabetical) |
|-------|--------|----------------------|
| Fractured | `fractured/` | 0 |
| Not fractured | `not fractured/` | 1 |

Class indices are inferred automatically by `tf.keras.utils.image_dataset_from_directory` (alphabetical order). The loader also accepts `non fractured/` as an alias for the negative class.

## Directory Layout

```
data/
  train/
    fractured/
      *.jpg, *.jpeg, *.png
    not fractured/
      *.jpg, *.jpeg, *.png
  test/
    fractured/
      *.jpg, *.jpeg, *.png
    not fractured/
      *.jpg, *.jpeg, *.png
  archive/
    ... (original 90-image Colab dataset)
```

## Preprocessing Pipeline

| Step | Details |
|------|---------|
| Load | `image_dataset_from_directory` |
| Resize | 1560 × 1170 (native dimensions from notebook) |
| Color mode | grayscale (1 channel) |
| Normalization | Rescaling layer: pixel values / 255.0 |
| Augmentation (train only) | RandomFlip horizontal, RandomRotation 5% |
| Augmentation (eval) | None |

## Split Strategy

| Split | Count | Method |
|-------|-------|--------|
| Train | 6,981 | 80% of `data/train/` (`validation_split=0.2`, `subset="training"`, `seed=123`) |
| Validation | 1,745 | 20% of `data/train/` (`subset="validation"`, `seed=123`) — used during training for early stopping |
| Test | 506 | Held-out `data/test/` — used for final evaluation (`python -m src.evaluate`) |

## Ethics and Privacy

- Images appear to be de-identified; verify no PII before sharing.
- Do not commit raw images to version control (`data/` is gitignored).
- This dataset is for educational research only.

## Known Data Limitations

- **Unknown provenance:** Source hospitals, patient demographics, and body regions not fully documented.
- **High resolution:** Slow training; may not reflect low-resource scanner quality in underserved clinics.
- **No geographic diversity metadata:** Generalizability to global populations unknown.

## Version Control Policy

Raw images and model weights are **not stored in Git**. The repository contains code, documentation, and small presentation assets only.

| Path | In Git? | Notes |
|------|---------|-------|
| `data/train/`, `data/test/` | No | Copy locally or download from backup (see [Dataset Setup](#dataset-setup)) |
| `data/.gitkeep` | Yes | Preserves empty folder structure |
| `outputs/checkpoints/*.h5` | No | Train locally or restore from backup |
| `outputs/training.log`, metrics | No | Regenerated by train/evaluate |

This keeps the GitHub repo small and avoids publishing large medical image files without an explicit license and review.

## Dataset Setup

Follow these steps after cloning [github.com/ishanvenkat/MedicalImaging](https://github.com/ishanvenkat/MedicalImaging).

### Step 1 — Create the folder structure

```bash
mkdir -p data/train/fractured data/train/"not fractured"
mkdir -p data/test/fractured data/test/"not fractured"
```

On Windows (PowerShell):

```powershell
New-Item -ItemType Directory -Force -Path data/train/fractured, "data/train/not fractured", data/test/fractured, "data/test/not fractured"
```

### Step 2 — Copy or download images

Place X-ray files (`.jpg`, `.jpeg`, `.png`) into the class folders. Sources:

| Source | When to use |
|--------|-------------|
| **Local backup** | You already have `data/train/` and `data/test/` from a previous training run |
| **Cloud backup** | Upload/download via Google Drive, OneDrive, or Dropbox — add your share link below |
| **`data/archive/`** | Reorganize from `Bone_Fracture_Dataset` into the train/test layout |

**Cloud download link (maintainer):** [TBD — add Google Drive or other share URL here]

GIF files mislabeled as `.jpg` are skipped automatically by the loader (6 known bad files in the full dataset).

### Step 3 — Verify counts

From the project root:

```bash
python -c "from src.data.dataset import discover_samples; from src.config import get_config; c=get_config(); print('train', len(discover_samples(c.train_dir))); print('test', len(discover_samples(c.test_dir)))"
```

| Split | Expected count |
|-------|----------------|
| `data/train/` | 8,723 |
| `data/test/` | 500 |

Counts may differ slightly if you add/remove images; training will still run if both classes are present in `data/train/`.

### Step 4 — (Optional) Restore model checkpoint

Checkpoints are not in Git. To evaluate without retraining:

1. Copy `model_weights.weights.h5` into `outputs/checkpoints/`
2. Run `python -m src.evaluate`

To train from scratch instead:

```bash
python -m src.train
python -m src.evaluate
```

Training at full resolution (1560×1170) on CPU takes several hours for 15 epochs.

### Step 5 — Custom data paths

Set environment variables to point elsewhere:

| Variable | Default |
|----------|---------|
| `MEDIMG_TRAIN_DIR` | `data/train` |
| `MEDIMG_TEST_DIR` | `data/test` |
| `MEDIMG_DATA_DIR` | `data` |

Example:

```bash
export MEDIMG_TRAIN_DIR=/mnt/datasets/fracture/train
export MEDIMG_TEST_DIR=/mnt/datasets/fracture/test
python -m src.train
```

### Alternative storage (not in Git)

For sharing the full dataset with collaborators:

- **Google Drive / OneDrive** — Simple; link in this file or README
- **Kaggle Datasets** — Good for public ML datasets with versioning
- **Hugging Face Datasets** — Good for portfolio projects and demos
- **Git LFS** — Not recommended for ~9k full-resolution images (quota and cost)
